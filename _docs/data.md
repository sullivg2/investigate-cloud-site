To date, most of the data about cloud provider sites has been collected manually. Here is a list of sources that were used for the initial dataset.

## Alibaba
* [Alibaba Cloud Regions & Zones](https://www.alibabacloud.com/help/doc-detail/40654.htm)

## Amazon Web Services
* [AWS Regions and Avilability Zones](https://aws.amazon.com/about-aws/global-infrastructure/regions_az/)
* [AWS Status Page](http://status.aws.amazon.com/rss/all.rss)

## Microsoft Azure
* [Azure Locations](https://azure.microsoft.com/en-us/global-infrastructure/locations/)
* [Azure Status Page](https://azurestatuscdn.azureedge.net/en-us/status/feed/)

## Digitalocean
* [Digitalocean Regional Availability Matrix](https://www.digitalocean.com/docs/platform/availability-matrix/)
* [DO Status Page](https://status.digitalocean.com/history.rss)

## Google Cloud Platforms
* [GCP Cloud Locations](https://cloud.google.com/about/locations/)
* [GCP Status Page](https://status.cloud.google.com/feed.atom)

## IBM Cloud
* [IBM Cloud Global Datacenters](https://www.ibm.com/cloud/data-centers/)
* [IBM Cloud Status Page](https://cloud.ibm.com/status/api/notifications/feed.rss)

## Oracle Cloud
* [Oracle Cloud Regions and Availability Domains](https://docs.cloud.oracle.com/en-us/iaas/Content/General/Concepts/regions.htm)
* [Oracle Cloud Status Page](https://ocistatus.oraclecloud.com/history.rss)

## Packet
* [Packet Cloud Locations](https://www.packet.com/cloud/locations/)
* [Packet Status Page](https://status.packet.com/history.rss)

## Tencent
* [Tencent Cloud Regions and Availability Zones](https://intl.cloud.tencent.com/document/product/213/6091)
